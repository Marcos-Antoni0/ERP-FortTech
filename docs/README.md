# Documentação do Projeto ERP FortTech

Bem-vindo à documentação técnica do projeto **ERP FortTech**.

Este guia tem como objetivo fornecer uma visão clara sobre a arquitetura, os padrões de desenvolvimento e os *guidelines* de design utilizados no projeto, facilitando a integração de novos desenvolvedores e garantindo a consistência do código.

## Índice
1.  [**Arquitetura Técnica**](Arquitetura%20T%C3%A9cnica.md)
    *   **Descrição:** Detalha a *stack* tecnológica (Django, Tailwind, PostgreSQL), o modelo de *Multi-Tenancy* e a estrutura de *apps* do projeto.
2.  [**Padrões de Código**](Padr%C3%B5es%20de%20C%C3%B3digo.md)
    *   **Descrição:** Define os *guidelines* de estilo de código, incluindo a aderência à PEP 8, o uso de aspas simples, a priorização de *Class Based Views* (CBVs) e a organização de *signals*.
3.  [**Design System**](Design%20System.md)
    *   **Descrição:** Apresenta a paleta de cores oficial, os padrões de componentes (botões, formulários, layouts) e o uso do Tailwind CSS para garantir a identidade visual e a responsividade.
4.  [**Lista de Tarefas**](14.%20Lista%20de%20Tarefas.md)
    *   **Descrição:** Contém o planejamento detalhado das *sprints*, com tarefas e subtarefas enumeradas, escopo de implementação e *checklists* para acompanhamento do desenvolvimento.

---

**Nota:** Esta documentação é o ponto de referência para qualquer desenvolvimento no projeto **ERP FortTech**.
